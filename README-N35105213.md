# JP Answers Extraction PoC - Project 6

**Purpose**: Automated batch extraction of legal questions through Employment Insurance (EI) Jurisprudence Portal (JP) system  
**Location**: EVA-JP-v1.2/docs/eva-foundation/projects/06-JP-Auto-Extraction  
**Status**: Production Ready - Professional v2.0  
**Encoding Policy**: ASCII-only for Windows enterprise compatibility

## What This Project Does

The extraction automation processes batches of legal questions through the JP Search Tool to extract case citations, legal precedents, and decision analysis. The system authenticates with Microsoft credentials, submits questions to the JP web interface, monitors multi-phase response processing, and exports structured results with comprehensive audit trails.

### Core Capabilities
- Batch processing of CSV question files through web automation
- Persistent session management with 24-hour authentication storage (eliminates authentication loops)
- Multi-phase JP response detection (Thinking -> Search Agent -> Document Analysis -> Summary Agent -> Complete)
- Professional case citation extraction with metadata
- Comprehensive evidence collection and audit trails
- Automated validation against 19 acceptance criteria
- Graceful error handling with context-rich diagnostics

## Input/Output Specifications

### Input Format
**File**: input/questions.csv  
**Columns**: question_id (required), question (required)  
**Encoding**: UTF-8 with BOM support  
**Sample Content**:
```
question_id,question
q001,which cases discuss antedate with "ignorance of the law"
q002,Are there cases where it's discussed whether vacation pay is considered earnings?
q003,show me a case where the client was sick but the appeal was denied
```
**Question Types**: Legal questions about Employment Insurance cases, appeals, eligibility decisions, and jurisprudence precedents

#### CSV Format Requirements
**User Responsibility**: Ensure proper CSV formatting before execution

**Comma Handling**: Questions containing commas must be properly quoted:
```
# CORRECT - Quoted text with commas
q020,"can you show me a case where the client wilfully disregarded the employer's condition for attendance, which led to dismissal?"

# INCORRECT - Unquoted commas break parsing
q020,can you show me a case where the client wilfully disregarded the employer's condition for attendance, which led to dismissal?
```

**Validation**: Test CSV format before execution:
```
python -c "import pandas as pd; df = pd.read_csv('input/questions.csv'); print(f'File valid: {len(df)} rows')"
```

**Character Encoding**: Save CSV files as UTF-8 to prevent encoding issues

### Output Formats
**Primary**: output/jp_answers_{timestamp}.csv with columns for question metadata, response content, case citations, processing phases, and timing metrics  
**Secondary**: output/jp_answers_{timestamp}.json for structured data consumption  
**Evidence**: debug/screenshots/ and debug/html/ for visual verification  
**Logs**: logs/jp_execution_{timestamp}.log with comprehensive operation details

**Success Indicators**: Look for case citations like "2024 FC 679" or "2021 SST 188", decision statements starting with "Decision:", and case links in the output files. Processing takes 3-8 minutes per complex legal question.

### Response Structure
Each processed question generates structured data including:
- Original question metadata (ID, text, timestamp)
- JP response phases (thinking, search, analysis, summary)
- Case citations with court references and decision links
- Processing timing and completion detection metrics
- Error context and recovery information when applicable

### Understanding Execution Output
**Session Information**: Each run creates unique session ID (e.g., `jp_session_20260123_085555`)

**Processing Phases**: JP responses go through multiple phases:
1. **"Thinking..."** - Initial query processing
2. **"Search Agent Federal Court of Appeals"** - Case law search
3. **"Document analysis, this may take additional time"** - Content processing  
4. **"Summary Agent"** - Results formatting
5. **Final formatted response** - Complete with case citations

**Completion Indicators**: Look for case citations ("2024 FC 679", "2021 SST 188"), decision statements ("Decision:"), and case links in output

**Timing Expectations**: Complex legal questions require 3-8 minutes each. Total batch time depends on question count and complexity.

## System Components

**jp_evidence_collector.py** - Compliance evidence system capturing screenshots, performance metrics, execution traces, and generating compliance reports for audit purposes

### Supporting Infrastructure

The system implements async/await architecture throughout with context managers for resource cleanup, retry logic with exponential backoff, and comprehensive logging with structured data output. Browser sessions use Playwright automation with session reuse, element waiting patterns, and response completion detection through content analysis.

## Common Execution Issues

### CSV Parsing Errors
**Problem**: "ParserError: Error tokenizing data" or "Expected 2 fields, saw 3"
**Cause**: Unquoted commas in question text
**Solution**: Add quotes around questions containing commas
**Validation**: `python -c "import pandas as pd; pd.read_csv('input/questions.csv')"`

### File Not Found Errors  
**Problem**: "FileNotFoundError: input/questions.csv"
**Cause**: Running from wrong directory
**Solution**: Use `jp_project_runner.py` (auto-detects directory) or run from project root
**Check**: `ls input/questions.csv` should show the file

### Parameter Syntax Issues
**Problem**: Unknown arguments or incorrect flags
**Cause**: Using wrong parameter names (--in vs --input)
**Solution**: Use documented parameters: `--input`, `--output`, `--headed`, `--limit`, `--connect`
**Reference**: `python scripts/jp_automation_main.py --help`

### Encoding Crashes (Windows)
**Problem**: UnicodeEncodeError with special characters
**Cause**: Windows cp1252 default encoding
**Solution**: Always set `PYTHONIOENCODING=utf-8` or use `jp_project_runner.py`

## Troubleshooting

### Encoding Issues (Windows Enterprise)
**Problem**: UnicodeEncodeError crashes in cp1252 environments  
**Solution**: Always run with PYTHONIOENCODING=utf-8 environment variable  
**Implementation**: Use run_jp_with_encoding.bat or set encoding manually

### Authentication Failures
**Problem**: Microsoft login timeouts, credential rejection, or stuck authentication loops  
**Solution**: The system uses persistent session management to avoid re-authentication  
**Implementation**: 
- Run `authenticate_jp.py` once to establish session (persists 24 hours)
- System automatically reuses saved session state for subsequent runs
- Use `analyze_jp_authentication.py` to diagnose and clean up session issues
- Enhanced session validation prevents authentication loops that previously caused timeouts

### Response Detection Timeouts
**Problem**: JP system taking longer than default 30-second timeout  
**Solution**: Complex legal queries require 3-8 minutes processing time  
**Implementation**: System polls every 5 seconds with content change detection

### Browser Automation Issues
**Problem**: Playwright browser launch failures or element detection problems  
**Solution**: Run jp_acceptance_tester.py to validate browser setup  
**Implementation**: Automated browser installation and configuration validation

### Network Connectivity
**Problem**: JP system unreachable or intermittent connection issues  
**Solution**: Network diagnostic checks and retry logic with backoff  
**Implementation**: Built-in connectivity validation and recovery patterns

### Output Validation Failures
**Problem**: Empty responses or incomplete case citation extraction  
**Solution**: Multi-phase completion detection ensures full response capture  
**Implementation**: Content analysis validates minimum response criteria and case citation patterns
